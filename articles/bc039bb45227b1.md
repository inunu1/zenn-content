# 1. はじめに（Introduction）

逐次計算モデルは、時系列データを前から順に処理することで状態を更新する手法であり、  
Elo レーティングはその典型例である。  
しかし、実際のアプリケーションでは過去のデータが後から修正・追加されることが多く、  
逐次更新では整合性が保てないという課題がある。

この問題は、レーティングのように「過去 → 現在 → 未来」が連鎖するモデルにおいて特に顕著である。  
途中のデータが変わると、その後のすべての計算結果が変化するため、  
部分的な更新では正しい状態を維持できない。

本記事では、この逐次性を前提としたうえで、  
**フル再計算を高速に行い、差分のみをバルク UPDATE する手法**を提示する。  
これにより、整合性を保ちながら高速に再計算できる仕組みを構築する。

本記事の貢献は以下の通りである。

1. Elo レーティングの逐次性がバッチ処理に与える影響を整理した。  
2. 全件再計算を O(N) で実現するメモリ逐次更新手法を提示した。  
3. 差分抽出とバルク UPDATE により、DB I/O を最小化する実装を示した。  
4. 実測により、約 300 件の対局データを 50ms 未満で再計算可能であることを示した。
